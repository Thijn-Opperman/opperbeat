## Onderzoeksvragen

### Primaire Onderzoeksvragen

**Hoofdvraag:**
> "Welke behoeften hebben DJ's op het gebied van muziekanalyse en organisatie, en hoe kunnen deze het beste worden ondersteund door een digitale tool?"

**Deelvragen:**

#### 1. Workflow & Proces Vragen

**RQ1.1:** Hoe bereiden DJ's zich voor op een optreden of set?  
**Sub-vragen:**
- Welke stappen nemen ze in de voorbereiding?
- Hoe organiseren ze hun muziekcollectie?
- Hoeveel tijd besteden ze aan voorbereiding?
- Wat zijn de meest tijdrovende onderdelen?

**RQ1.2:** Wat is de rol van BPM en key informatie in het DJ-proces?  
**Sub-vragen:**
- Hoe vaak gebruiken ze BPM/key informatie tijdens het mixen?
- In welke fase van het proces is deze informatie het meest belangrijk?
- Hoe nauwkeurig moet deze informatie zijn?
- Vertrouwen ze op automatische detectie of handmatige invoer?

#### 2. Tool Gebruik & Ervaring Vragen

**RQ2.1:** Welke tools gebruiken DJ's momenteel voor muziekanalyse en organisatie?  
**Sub-vragen:**
- Welke software gebruiken ze (Serato, Traktor, Rekordbox, etc.)?
- Waarom hebben ze voor deze tools gekozen?
- Wat zijn de sterke en zwakke punten van deze tools?
- Gebruiken ze meerdere tools naast elkaar?

**RQ2.2:** Wat zijn de belangrijkste pain points bij het gebruik van bestaande tools?  
**Sub-vragen:**
- Welke frustraties ervaren ze dagelijks?
- Wat kost hen het meeste tijd?
- Welke functionaliteiten missen ze?
- Wat zou hun workflow significant verbeteren?

#### 3. Behoeften & Wensen Vragen

**RQ3.1:** Welke functionaliteiten zijn essentieel voor DJ's in een muziekanalyse tool?  
**Sub-vragen:**
- Wat moet een tool minimaal kunnen?
- Welke features zijn "nice to have" vs "must have"?
- Wat zou hen overtuigen om over te stappen naar een nieuwe tool?
- Zijn er features waarvan ze niet wisten dat ze ze nodig hadden?

**RQ3.2:** Wat zijn de belangrijkste criteria bij het kiezen van een muziekanalyse tool?  
**Sub-vragen:**
- Wat is belangrijker: snelheid, nauwkeurigheid, gebruiksgemak, of prijs?
- Hoe belangrijk is cloud-synchronisatie?
- Is mobile access belangrijk?
- Wat vinden ze van betaalde vs gratis tools?

#### 4. Technologie & Platform Vragen

**RQ4.1:** Welke devices en platforms gebruiken DJ's voor hun werk?  
**Sub-vragen:**
- Werken ze op desktop, laptop, tablet, of mobile?
- Welk besturingssysteem gebruiken ze?
- Gebruiken ze meerdere devices tijdens het DJ-en?
- Is cross-platform synchronisatie belangrijk?

**RQ4.2:** Hoe belangrijk is cloud-backup en synchronisatie?  
**Sub-vragen:**
- Maken ze regelmatig backups van hun muziekcollectie?
- Hebben ze wel eens data verloren?
- Zouden ze hun collectie in de cloud willen opslaan?
- Wat zijn hun privacy concerns rondom cloud-opslag?

#### 5. Usability & User Experience Vragen

**RQ5.1:** Wat maakt een interface gebruiksvriendelijk voor DJ's?  
**Sub-vragen:**
- Wat verwachten ze qua navigatie en structuur?
- Hoe belangrijk is visueel design vs functionaliteit?
- Wat zijn hun verwachtingen rondom snelheid en performance?
- Wat frustreert hen het meest in digitale interfaces?

**RQ5.2:** Welke informatie moet prominent getoond worden tijdens het mixen?  
**Sub-vragen:**
- Welke metadata is het belangrijkst (BPM, key, genre, etc.)?
- Hoe moeten deze gevisualiseerd worden?
- Wanneer hebben ze deze informatie nodig (voorbereiding vs live)?
- Wat is information overload voor hen?

### Secundaire Onderzoeksvragen

**RQ6:** Hoe verschillen de behoeften tussen verschillende types DJ's?  
- Club DJ vs Radio DJ vs Wedding DJ
- Beginner vs Expert
- Hobbyist vs Professional

**RQ7:** Wat is de rol van community en social features?  
- Delen DJ's graag playlists?
- Zouden ze van anderen willen leren?
- Hoe belangrijk is het om te zien wat anderen doen?

**RQ8:** Wat zijn de toekomstverwachtingen voor DJ-tools?  
- Welke trends zien ze aankomen?
- Welke technologie zou hun werk revolutioneren?
- Wat verwachten ze van AI/ML in DJ-tools?

## Interview Structuur

### Interview Format

**Type:** Semi-gestructureerd interview  
**Duur:** 45-60 minuten  
**Setting:** Online (Zoom/Teams) of fysiek  
**Recording:** Met toestemming van deelnemer

### Interview Flow

1. **Opening & Warm-up** (5 minuten)
   - Introductie
   - Context uitleg
   - Consent voor opname

2. **Workflow & Process** (15 minuten)
   - Huidige workflow
   - Proces beschrijving
   - Tijdsbesteding

3. **Tool Gebruik & Pain Points** (15 minuten)
   - Huidige tools
   - Frustraties
   - Missende features

4. **Behoeften & Wensen** (10 minuten)
   - Ideale tool
   - Must-have features
   - Criteria

5. **Opperbeat Specifiek** (10 minuten)
   - Prototype feedback
   - Features evaluatie
   - Suggesties

6. **Afsluiting** (5 minuten)
   - Open vragen
   - Contact gegevens
   - Dankwoord

## Data Analyse Framework

### Na het Interview

#### 1. Transcriptie & Notities
- [ ] Interview opname transcriberen
- [ ] Key quotes markeren
- [ ] Observaties noteren
- [ ] Timing van belangrijke momenten

#### 2. Coderen & Categoriseren

**Codes te gebruiken:**

**Workflow Codes:**
- `WORKFLOW_PREP` - Voorbereidingsproces
- `WORKFLOW_LIVE` - Live DJ process
- `WORKFLOW_ORG` - Organisatie methoden

**Tool Codes:**
- `TOOL_CURRENT` - Huidige tools
- `TOOL_PAIN` - Pain points
- `TOOL_MISSING` - Missende features

**Need Codes:**
- `NEED_MUST` - Must-have features
- `NEED_NICE` - Nice-to-have features
- `NEED_WISH` - Wishlist features

**User Type Codes:**
- `USER_BEGINNER` - Beginner DJ
- `USER_EXPERT` - Expert DJ
- `USER_PROFESSIONAL` - Professional DJ

#### 3. Pattern Analysis

**Zoek naar:**
- **Frequentie:** Hoe vaak wordt een thema genoemd?
- **Intensiteit:** Hoe emotioneel is de reactie?
- **Consensus:** Zijn er verschillen tussen gebruikers?
- **Conflicten:** Zijn er tegenstrijdige behoeften?

#### 4. Insight Extraction

**Voor elke interview:**

**Belangrijkste Inzichten:**
1. [ ] Top 3 pain points
2. [ ] Top 3 must-have features
3. [ ] Top 3 onverwachte inzichten
4. [ ] Werkwijze verschillen/overeenkomsten

**Opperbeat Specifiek:**
1. [ ] Features die goed werken
2. [ ] Features die verbetering nodig hebben
3. [ ] Missende features
4. [ ] Usability issues

#### 5. Persona Updates

**Gebruik interviews om personas te verbeteren:**
- [ ] Persona behoeften bijwerken
- [ ] Pain points toevoegen
- [ ] Quotes toevoegen aan personas
- [ ] Scenario's verfijnen

#### 6. Product Prioriteiten

**Output voor Product Team:**

**Prioriteit Matrix:**
```
| Impact | Hoge Impact | Lage Impact |
|--------|-------------|-------------|
| Makkelijk | Quick Wins | Fill-ins |
| Moeilijk | Strategic | Money Pits |
```

**Voorbeelden van categorisatie:**
- **Quick Win:** Duidelijke error messages (makkelijke UX verbetering, hoog impact)
- **Strategic:** Cloud synchronisatie (complex, maar essentieel)
- **Fill-in:** Custom themes (leuk, maar niet cruciaal)
- **Money Pit:** Advanced AI features (moeilijk, onzekere waarde)
